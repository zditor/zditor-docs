# 元信息（MetaData）

## 概述

元信息是 Zditor 中用于在文章顶部展示文章基本信息的组件，包括标题、副标题、作者、创建时间、标签等。元信息以 [Front Matter](https://jekyllrb.com/docs/front-matter/) 格式存储在文档开头。

## 支持的字段

|字段名 |类型 |说明 |示例 |
|---|---|---|---|
|`title` |string |文章标题 |`我的第一篇文章` |
|`subtitle` |string |文章副标题 |`一篇关于技术分享的文章` |
|`authors` |array |作者列表（支持多个作者） |`["张三", "李四"]` |
|`author_avatar` |string |作者头像图片路径 |`./images/avatar.jpg` |
|`createdAt` |string |创建时间（ISO 8601 格式） |`2024-01-15T10:30:00` |
|`tags` |array |标签列表 |`["技术", "分享", "前端"]` |
|`background` |string |背景图片路径或URL |`./images/bg.jpg` |
|`tone` |string |主题色调（影响颜色风格） |`cold` |
|`show_property` |string |是否显示属性面板（设为 "true" 显示） |`true` |

## 基础用法

![元信息示意图](assets/截屏2025-11-02_19.54.18.png)

### Front Matter 基本格式

```yaml
---
title: 文章标题
subtitle: 文章副标题
authors:
  - 作者姓名
createdAt: 2024-01-15T10:30:00
tags:
  - 标签1
  - 标签2
background: ./images/background.jpg
author_avatar: ./images/avatar.jpg
tone: cold
show_property: true
---
```

### 仅设置必要字段

```yaml
---
title: 简洁的文章
authors:
  - 作者名
tags:
  - 技术
---
```

## 详细字段说明

### title（标题）

- **类型**: string
- **必填**: 否
- **说明**: 文章的主要标题，显示在元信息区域的中上部
- **特点**: 支持中英文，超长时自动换行

### subtitle（副标题）

- **类型**: string
- **必填**: 否
- **说明**: 文章的副标题，标题下方显示
- **特点**: 可以为空，用于补充说明文章主题

### authors（作者列表）

- **类型**: array of string
- **必填**: 否
- **说明**: 文章作者列表，支持多个作者
- **显示规则**:
  - 第一个作者显示在用户信息区域
  - 超过 1 个作者时，最多显示到第 3 个作者，其余显示为 "等X人"
- **示例**:

  ```yaml
  authors:
    - 张三
    - 李四
    - 王五
    - 赵六
  # 显示: "张三" 和 "李四, 王五 等3人"
  ```

### author_avatar（作者头像）

- **类型**: string
- **必填**: 否
- **说明**: 作者头像图片路径，支持相对路径和绝对路径
- **特点**:
  - 当设置时，显示头像图片
  - 未设置但有作者信息时，显示作者姓名的首字母
  - 支持多种图片格式（jpg, png, gif 等）

### createdAt（创建时间）

- **类型**: string
- **必填**: 否
- **格式**: ISO 8601 标准（建议使用 `YYYY-MM-DDTHH:mm:ss` 格式）
- **显示**: 格式化为本地化的长日期格式（如：2024年1月15日）
- **示例**: `2024-01-15T10:30:00` → 显示为 "2024年1月15日"

### tags（标签）

- **类型**: array of string
- **必填**: 否
- **说明**: 文章标签列表，用于分类和检索
- **显示**: 以 `#标签名` 的形式显示，带有背景色的胶囊样式
- **特点**: 支持横向滚动，超长时不换行

### background（背景图片）

- **类型**: string
- **必填**: 否
- **说明**: 背景图片路径或 URL
- **特点**:
  - 图片自动适配容器大小
  - 支持本地路径和远程 URL
  - 文字覆盖在背景图上方，带有渐变遮罩以保证可读性

### tone（主题色调）

- **类型**: string
- **必填**: 否
- **说明**: 控制元信息区域的主题色调
- **当前支持**: `cold`（冷色调）
- **影响**: 文字颜色、背景色、UI 组件的颜色风格

### show_property（显示属性面板）

- **类型**: string
- **必填**: 否
- **说明**: 是否在元信息下方显示详细的属性面板
- **取值**:
  - `"true"`: 显示属性面板
  - 其他值或空: 不显示

## 编辑功能

### 悬停工具栏

当鼠标悬停在元信息区域时，会显示工具栏，包含以下功能：

- **编辑元数据**: 打开元数据编辑模态框
- **更换背景**: 打开图片选择器，更换背景图片
- **删除元信息**: 移除当前文章的元信息

### 编辑方式


1. **直接编辑 Front Matter**: 在文档开头修改 YAML 内容
2. **使用工具栏**: 悬停后点击编辑按钮，通过图形界面编辑

## 样式和主题

元信息组件采用现代化的设计风格：

- **背景**: 可选的背景图片，带有渐变遮罩
- **文字层次**:
  - 创建时间：次要信息，较小字号，浅色
  - 标题：主要标题，大字号，粗体
  - 副标题：辅助信息，常规字号，半透明
  - 作者信息：中等字号
  - 标签：胶囊样式，带有 `#` 符号
- **布局**: 分为上下两部分
  - 上半部分：创建时间、标题、副标题
  - 下半部分（底部模糊区域）：作者信息和标签
- **交互效果**:
  - 悬停时显示工具栏
  - 标签悬停时轻微放大
  - 图片懒加载

## 注意事项


1. **Front Matter 格式**: 元信息必须位于文档开头，使用 `---` 包围
2. **文件路径**: 使用相对路径时，路径相对于当前文档所在目录
3. **时间格式**: 建议使用 ISO 8601 格式，确保跨平台兼容性
4. **作者信息**: `authors` 和 `author_avatar` 可以配合使用，也可以单独使用
5. **性能**: 背景图片和头像采用懒加载方式，不影响页面初始加载速度
6. **响应式**: 自动适配不同屏幕尺寸，移动端友好
