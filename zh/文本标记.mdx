Zditor 同时支持标准行内格式和更适合评审场景的扩展标记。你可以用粗体、斜体、高亮、行内代码组织正文，也可以用注释和修订把说明与修改建议直接写进文档。

## 基础行内标记

| 标记 | 语法 | 常见用途 |
| --- | --- | --- |
| 粗体 | `**文本**` | 关键结论、重点术语 |
| 斜体 | `*文本*` | 轻度强调、外来词、特殊术语 |
| 高亮 | `==文本==` | 定义、重点摘录、重要提醒 |
| 行内代码 | `` `code` `` | 命令、变量、标识符 |
| 链接 | `[文本](链接)` | 跳转和引用 |

<Tip>
  文本标记的作用是帮助读者更快理解结构，而不是把一段话变成样式堆叠区。
</Tip>

## 评审型标记

<AccordionGroup>
  <Accordion title="注释：mode=tip" icon="lightbulb" defaultOpen>
    当原文本身没问题、只是需要补充背景时，用注释最合适。

    ```md
    [title](src|mode=tip)
    ```

    完整写法示例：

    ```md
    [术语](这里写说明|mode=tip|style=teal)
    ```

    适合场景：

    - 解释术语
    - 补充上下文
    - 给读者留阅读提示
  </Accordion>

  <Accordion title="修订：mode=revision" icon="pen-to-square">
    当你希望原文被替换成更合适的说法时，用修订。

    ```md
    [title](src|mode=revision)
    ```

    完整写法示例：

    ```md
    [原文](这里写为什么要改|mode=revision|style=indigo|advice=修改后的文本)
    ```

    适合场景：

    - 修正错别字
    - 优化措辞
    - 把模糊时间改成明确时间
  </Accordion>

  <Accordion title="什么时候要把空格写成 %20" icon="code">
    如果你直接手写 Markdown 源码，`tip` 和 `revision` 括号里的空格需要编码成 `%20`。

    这适用于：

    - 说明文字
    - 文件路径
    - URL
    - `advice` 的值
  </Accordion>
</AccordionGroup>

## 颜色建议

| 颜色 | 适合场景 |
| --- | --- |
| `red` | 错误、删除、时间模糊、拼写问题 |
| `indigo` | 措辞优化、风格统一、表达升级 |
| `teal` | 中性说明、术语解释、一般建议 |
| `yellow` | 提醒、注意事项、额外关注 |
| `green` | 正向说明、建议保留、通过确认 |

## 可直接复用的模板

### 注释模板

```md
这里有一个 [术语](这里解释它的含义|mode=tip|style=teal) 需要说明。
```

### 修订模板

```md
请把 [原来的说法](这里说明为什么要改|mode=revision|style=indigo|advice=修改后的说法) 换成更准确的表达。
```

<Warning>
  `revision` 必须带 `advice`。如果没有替换建议，这个标记只是在指出问题，没有给出可执行的改法。
</Warning>
