知识图谱是 Zditor 的创新功能之一，通过交互式力导向图将文件和它们之间的关系可视化呈现。本文档将帮助你充分利用这个强大的可视化工具来理解和导航你的项目结构。

## 功能概览

知识图谱使用基于物理模拟的力导向算法，自动将文件（节点）和它们之间的链接关系以图形方式展现。每个文件表示为一个彩色节点，文件之间的关联通过带箭头的连线显示。

## 链接语法解析

知识图谱通过解析 Markdown 文件中的链接语法自动构建文件关系网络。理解解析机制可以帮助你更好地组织和利用知识图谱。

### Markdown 链接语法

知识图谱识别标准的 Markdown 链接语法：

```markdown
[链接文本](文件路径)
```

<AccordionGroup>
  <Accordion title="链接组成部分" icon="link" defaultOpen>
    **链接文本（Title）**
    - 方括号 `[]` 中的内容
    - 作为图中连接线的标签显示
    - 描述两个文件之间的关系性质

    **文件路径（Path）**
    - 圆括号 `()` 中的内容
    - 指向目标文件的路径（相对或绝对）
    - 确定连接线的目标节点

    **示例**：
    ```markdown
    这是一个 [引用文档](./docs/reference.md) 的链接。
    这里 [依赖配置](../config/settings.json) 文件。
    ```
  </Accordion>

  <Accordion title="如何转换为图形元素" icon="diagram-project">
    每个 Markdown 链接会被解析为图中的一条**有向边**（带箭头的连接线）：

    - **源节点（Source）**：包含链接的 Markdown 文件
    - **目标节点（Target）**：链接指向的文件
    - **箭头方向**：从源节点指向目标节点
    - **连接线标签**：显示链接文本（可选）

    **可视化示例**：

    如果文件 `README.md` 包含链接 `[使用说明](docs/guide.md)`，则图中会显示：

    ```
    README.md ----[使用说明]----> docs/guide.md
    ```
  </Accordion>

  <Accordion title="箭头方向的含义" icon="arrow-right">
    箭头方向表示**引用关系**：

    - **A → B**：文件 A 引用（链接到）文件 B
    - **A ← B**：文件 B 引用（链接到）文件 A
    - **A ⇄ B**：两个文件互相引用（双向链接）

    这种可视化让你可以快速识别：
    - **出度高的节点**：大量引用其他文件（内容聚合者）
    - **入度高的节点**：被大量引用（核心资源）
    - **双向关系**：紧密相关的文件对
  </Accordion>
</AccordionGroup>

### 多重链接处理

当两个文件之间存在多条链接时，知识图谱会智能处理以避免视觉混乱。

<AccordionGroup>
  <Accordion title="同向多重链接" icon="arrows-turn-right">
    如果文件 A 中多次链接到文件 B（使用不同的链接文本），每条链接会显示为**不同曲率的曲线**：

    ```markdown
    <!-- 在 A.md 中 -->
    [第一个引用](B.md)
    [第二个引用](B.md)
    [第三个引用](B.md)
    ```

    图中显示为：
    ```
    A ⟲ B  (曲率 0.2)
    A ⟳ B  (曲率 0.4)
    A ⟴ B  (曲率 0.6)
    ```

    每条曲线的曲率递增 0.2，确保所有链接都清晰可见。
  </Accordion>

  <Accordion title="双向链接" icon="arrows-left-right">
    当文件 A 和 B 互相链接时，使用不同曲率的曲线区分方向：

    ```markdown
    <!-- 在 A.md 中 -->
    [关联到 B](B.md)

    <!-- 在 B.md 中 -->
    [返回 A](A.md)
    ```

    图中显示为：
    ```
    A --[关联到 B]--> B
    A <--[返回 A]-- B
    ```

    双向链接使用 0.25 的曲率，让去程和回程的箭头分开显示。
  </Accordion>
</AccordionGroup>

### 解析规则与限制

<Warning>
  **当前版本的解析限制**：

  - 仅解析标准 Markdown 链接语法 `[text](path)`
  - 不支持 Wiki 链接语法 `[[page]]`（未来可能支持）
  - 不解析图片链接 `![alt](image.png)`（图片文件作为独立节点）
  - 不解析 HTML `<a>` 标签
  - 相对路径基于当前文件所在目录解析
</Warning>

<Tip>
  **最佳实践**：

  - 使用描述性的链接文本，让图中的连接线标签更有意义
  - 示例：用 `[API 参考](api.md)` 而不是 `[点击这里](api.md)`
  - 保持链接文本简洁（建议 2-5 个字），避免图中标签过长
  - 使用一致的命名风格，便于识别链接类型
</Tip>

### 实际应用示例

假设你有以下文件结构和链接关系：

```markdown
<!-- docs/README.md -->
本项目包含以下部分：
- [快速开始指南](./quickstart.md)
- [API 文档](./api.md)
- [常见问题](./faq.md)

<!-- docs/quickstart.md -->
开始前请阅读 [README](./README.md)
配置方法见 [API 文档](./api.md)

<!-- docs/api.md -->
参考 [README](./README.md) 了解概览
```

知识图谱会生成如下关系：

```
README.md ----[快速开始指南]----> quickstart.md
README.md ----[API 文档]--------> api.md
README.md ----[常见问题]--------> faq.md
quickstart.md ---[README]-------> README.md
quickstart.md ---[API 文档]-----> api.md
api.md --------[README]---------> README.md
```

在图中你可以直观地看到：
- `README.md` 是中心节点（被多次引用，也引用其他文件）
- `api.md` 是热门目标（多个文件链接到它）
- `README.md ⇄ quickstart.md` 和 `README.md ⇄ api.md` 存在双向引用

## 界面布局

知识图谱界面主要由三部分组成：

### 主画布区域

中央的大型画布区域显示所有节点和链接，支持自由缩放和平移。

### 控制面板

右侧（或可配置位置）的控制面板提供：
- 文件类型过滤器
- 搜索功能
- 配置选项
- 统计信息

### LOD 指示器

左下角显示当前的细节层次（Level of Detail）级别：
- **Level 0 (Minimal)** - 仅显示节点，不显示标签
- **Level 1 (Full Detail)** - 显示节点和标签

<Tip>
  **提示**：LOD 系统会根据缩放级别自动调整显示内容，确保在任何缩放级别都有良好的性能和可读性。
</Tip>

## 基础交互操作

知识图谱提供了丰富的交互方式，让你能够自由探索文件关系网络。

### 视图控制

<AccordionGroup>
  <Accordion title="平移画布" icon="hand">
    在画布空白区域点击并拖动，可以平移整个视图。
  </Accordion>

  <Accordion title="缩放画布" icon="magnifying-glass">
    使用鼠标滚轮或触控板的缩放手势来放大或缩小视图。

    - **放大**：向上滚动或双指外扩
    - **缩小**：向下滚动或双指内捏
  </Accordion>
</AccordionGroup>

<Tip>
  **提示**：当缩放级别达到 1.2 倍或更高时，节点标签会自动显示，方便你查看文件名。
</Tip>

### 节点交互

<AccordionGroup>
  <Accordion title="单击节点" icon="mouse-pointer" defaultOpen>
    单击节点会选中该节点，同时：
    - 节点显示白色外框
    - 与该节点相连的所有链接高亮显示（紫色）
    - 其他链接变为半透明

    这让你能够快速识别当前文件的所有关联关系。
  </Accordion>

  <Accordion title="双击节点" icon="hand-pointer">
    双击节点会在编辑器中打开对应的文件，这是快速导航到文件的便捷方式。
  </Accordion>

  <Accordion title="拖拽节点" icon="arrows-alt">
    点击并拖动节点可以手动调整其位置。松开鼠标后，节点会被固定在该位置（而不是继续随物理模拟移动）。

    这对于手动优化重要节点的布局非常有用。
  </Accordion>
</AccordionGroup>

### 链接交互

<AccordionGroup>
  <Accordion title="单击链接" icon="link">
    单击连接两个节点的链接线会选中该链接：
    - 链接线变粗并变为深紫色
    - 显示链接标题（如果有）

    这让你能够查看特定关系的详细信息。
  </Accordion>

  <Accordion title="查看链接标题" icon="tag">
    将鼠标悬停在链接上会显示工具提示，展示链接的标题或描述信息。
  </Accordion>
</AccordionGroup>

<Note>
  **关于链接方向**：画布中的箭头指示链接方向（从源到目标）。当两个文件之间有多条链接时，系统会自动使用不同的曲率显示它们，避免重叠。
</Note>

## 搜索功能

知识图谱提供强大的搜索功能，帮助你快速定位目标文件或链接。

### 如何搜索

1. 打开控制面板（如果未展开）
2. 在搜索框中输入关键词
3. 搜索结果会实时显示

### 搜索范围

搜索功能支持两种类型的搜索：

<AccordionGroup>
  <Accordion title="节点搜索" icon="circle-nodes">
    按文件名或路径搜索节点。

    **示例**：
    - 输入 `README` 查找所有包含 "README" 的文件
    - 输入 `src/` 查找 src 目录下的文件
  </Accordion>

  <Accordion title="链接搜索" icon="link">
    按链接标题或描述搜索关联关系。

    **示例**：
    - 输入链接的标题关键词
    - 搜索特定类型的引用关系
  </Accordion>
</AccordionGroup>

搜索结果会限制为最多 50 个匹配项，确保界面响应快速。

## 过滤功能

通过文件类型过滤器，你可以专注于特定类型的文件和它们的关系。

### 使用过滤器

1. 打开控制面板
2. 点击文件类型过滤器按钮
3. 选择要显示的文件类型

### 过滤器选项

- **All** - 显示所有类型的文件（默认）
- **Markdown** - 仅显示 Markdown 文件及其关联
- **Images** - 仅显示图片文件
- **Videos** - 仅显示视频文件
- **Code** - 仅显示代码文件
- **Documents** - 仅显示文档文件
- **Other** - 仅显示其他类型文件

<Tip>
  **提示**：可以同时选择多个过滤器。当选择多个类型时，只有这些类型的节点及它们之间的链接会显示。
</Tip>

<Warning>
  **注意**：应用过滤器后，只有被过滤类型之间的链接才会显示。如果一条链接的源或目标节点被过滤掉，该链接也不会显示。
</Warning>

## 配置选项

知识图谱提供了丰富的配置选项，让你可以根据偏好调整可视化效果。

### 显示设置

<AccordionGroup>
  <Accordion title="标签显示阈值" icon="text-size">
    控制在何种缩放级别下显示节点标签。

    - **默认值**：1.2
    - **调整建议**：如果你想在更低的缩放级别看到标签，可以降低此值
  </Accordion>

  <Accordion title="节点大小缩放" icon="circle">
    调整所有节点的显示大小。

    - **默认值**：1.0
    - **调整建议**：增大此值让节点更明显，减小此值获得更紧凑的布局
  </Accordion>

  <Accordion title="链接粗细" icon="minus">
    控制连接线的粗细程度。

    - **默认值**：1.0
    - **调整建议**：增大此值使链接更清晰可见
  </Accordion>

  <Accordion title="显示链接标签" icon="tags">
    切换是否在悬停时显示链接标题。

    - **默认值**：开启
    - **关闭时**：悬停链接不会显示工具提示，减少视觉干扰
  </Accordion>
</AccordionGroup>

### 物理模拟设置

这些设置控制节点自动布局的物理模拟行为。

<AccordionGroup>
  <Accordion title="节点斥力强度" icon="magnet">
    控制节点之间相互排斥的力度。

    - **默认值**：-200
    - **负值越大**：节点之间距离越远
    - **调整建议**：如果图形过于紧密，增大斥力（更负的值）
  </Accordion>

  <Accordion title="链接距离" icon="ruler">
    控制连接节点之间的理想距离。

    - **默认值**：50
    - **值越大**：连接的节点距离越远
    - **调整建议**：根据项目复杂度和个人偏好调整
  </Accordion>
</AccordionGroup>

<Note>
  **关于物理模拟**：画布使用 D3.js 的力导向图算法。修改物理参数后，系统会自动重新加热模拟，让节点按新参数重新布局。这个过程可能需要几秒钟才能稳定。
</Note>

## 统计信息

控制面板底部显示实时统计信息，帮助你了解当前图形的状态：

| 指标 | 说明 |
| --- | --- |
| **FPS** | 每秒帧数，反映渲染性能 |
| **Zoom** | 当前缩放级别 |
| **Visible** | 当前可见的节点数量 |
| **Nodes** | 总节点数（所有文件） |
| **Links** | 总链接数（所有关联关系） |

<Tip>
  **性能提示**：如果 FPS 较低（低于 30），可以尝试：
  - 使用过滤器减少显示的节点数
  - 关闭链接标签显示
  - 减小节点缩放比例
</Tip>

## 使用场景

<AccordionGroup>
  <Accordion title="理解新项目结构" icon="folder-tree">
    当接手一个新项目时，使用知识图谱可以快速了解：
    - 项目的整体架构
    - 核心文件和模块
    - 文件之间的依赖关系

    通过可视化，你可以在几分钟内对项目有宏观的认识。
  </Accordion>

  <Accordion title="重构前的影响分析" icon="code-branch">
    在修改某个文件之前：
    1. 在画布中选中该文件节点
    2. 查看所有高亮的链接
    3. 识别哪些文件会受到修改的影响

    这有助于避免意外破坏其他功能。
  </Accordion>

  <Accordion title="发现孤立文件" icon="unlink">
    没有任何链接的节点（孤立节点）可能是：
    - 未被使用的旧代码
    - 待集成的新功能
    - 独立的工具脚本

    定期检查可以帮助清理项目。
  </Accordion>

  <Accordion title="文档关系梳理" icon="book">
    对于文档项目（如知识库、笔记系统）：
    - 查看 Markdown 文件之间的引用关系
    - 发现相关主题的文章集群
    - 优化文档的链接结构
  </Accordion>
</AccordionGroup>

## 性能优化建议

处理大型项目（数百个文件）时，考虑以下优化策略：

<Steps>
  <Step title="使用文件类型过滤">
    不要一次显示所有文件。先用过滤器聚焦到你关心的文件类型。
  </Step>

  <Step title="调整物理参数">
    如果渲染缓慢，适当增大节点斥力和链接距离，减少节点重叠计算。
  </Step>

  <Step title="固定关键节点">
    手动拖拽并固定一些节点后，物理模拟的计算量会减少，性能会提升。
  </Step>

  <Step title="关闭链接标签">
    在配置中关闭"显示链接标签"可以减少渲染开销。
  </Step>
</Steps>

## 常见问题

<AccordionGroup>
  <Accordion title="为什么有些节点在画布外看不到？" icon="question">
    初始布局时，节点可能分散到画布外。尝试：
    - 缩小视图（向外滚动）
    - 在搜索框中搜索该文件，然后点击搜索结果
    - 等待物理模拟稳定（通常需要几秒钟）
  </Accordion>

  <Accordion title="如何重置节点的固定位置？" icon="question">
    目前知识图谱不支持直接重置单个节点。你可以：
    - 切换过滤器强制重新布局
    - 刷新整个图谱组件
  </Accordion>

  <Accordion title="双击节点打开文件没有反应？" icon="question">
    确保：
    - 文件路径正确存在
    - 两次点击间隔小于 300 毫秒
    - 没有在点击之间拖拽节点
  </Accordion>

  <Accordion title="链接箭头重叠难以区分？" icon="question">
    当两个文件之间有多条链接时，系统会自动使用不同的曲率显示。如果仍然重叠：
    - 放大视图查看细节
    - 单击节点高亮相关链接
    - 拖拽节点调整位置
  </Accordion>

  <Accordion title="如何导出或分享图谱视图？" icon="question">
    当前版本不支持直接导出图形。你可以：
    - 使用系统截图工具捕获画面
    - 记录当前的缩放和平移位置，分享坐标
  </Accordion>

  <Accordion title="物理模拟一直在晃动不稳定？" icon="question">
    这通常是因为力参数设置不当。尝试：
    - 增大链接距离
    - 减小节点斥力强度（绝对值）
    - 手动拖拽并固定一些节点作为"锚点"
  </Accordion>
</AccordionGroup>

## 键盘快捷键

虽然知识图谱主要依赖鼠标交互，但以下快捷键可以提升效率：

| 快捷键 | 功能 |
| --- | --- |
| **Esc** | 取消当前选择的节点或链接 |
| **空格 + 拖拽** | 快速平移画布（某些浏览器） |
| **Ctrl/Cmd + 滚轮** | 精细控制缩放速度 |

## 技术细节

<Note>
  知识图谱基于以下技术构建：

  - **react-force-graph-2d**：提供力导向图渲染引擎
  - **D3.js**：提供物理模拟算法
  - **Canvas API**：提供高性能的 2D 渲染
  - **自适应 LOD**：根据缩放级别动态调整细节显示

  所有渲染都在客户端进行，确保流畅的交互体验。组件使用 React.lazy 进行代码分割，优化初始加载时间。
</Note>
