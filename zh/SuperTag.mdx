SuperTag 让 Zditor 可以把 Markdown 文件理解成结构化记录，并把这些记录作为标签节点插入到其他文档中。

## 功能概览

<CardGroup cols={2}>
  <Card title="结构化记录" icon="database">
    frontmatter 负责元数据，正文仍然保持普通 Markdown 写作方式。
  </Card>

  <Card title="共享 class" icon="table">
    多个文件只要共享同一个 `class`，就会被聚合成同一张逻辑表。
  </Card>

  <Card title="内联引用" icon="link">
    用 `mode=supertag` 在文档里插入结构化记录引用。
  </Card>

  <Card title="卡片布局字段" icon="table">
    用 `col`、`row`、`cover`、`warm` 等字段控制展示方式。
  </Card>
</CardGroup>

## 核心概念

<AccordionGroup>
  <Accordion title="一条记录本质上仍然是 Markdown 文件" icon="file" defaultOpen>
    SuperTag 记录本质上就是一篇带类型化 frontmatter 的 Markdown 文档。

    ```yaml
    ---
    title:
      type: text
      label: 片名
      value: "教父"
    director:
      type: text
      label: 导演
      value: "弗朗西斯·福特·科波拉"
    year:
      type: number
      label: 年份
      value: 1972
    class:
      type: text
      label: class
      value: "ClassicMovie"
    ---
    ```
  </Accordion>

  <Accordion title="class 字段决定归组方式" icon="table">
    只要多个文件的 `class` 值相同，Zditor 就会把它们理解成同一类结构化记录。

    ```yaml
    class:
      type: text
      label: class
      value: "ClassicMovie"
    ```
  </Accordion>

  <Accordion title="col 和 row 控制卡片布局" icon="table">
    这些布局字段决定哪些元数据会出现在卡片主区域和辅助区域。

    ```yaml
    col:
      type: array
      label: Col
      value: [year, title, director]
    row:
      type: array
      label: Row
      value: [genre, rank, score, watched]
    ```

    - `col` 适合纵向主信息区
    - `row` 适合底部或辅助信息区
  </Accordion>
</AccordionGroup>

## 引用语法

使用带 `mode=supertag` 的普通 Markdown 链接：

```markdown
[title](src|mode=supertag)
```

<Note>
  产品里公开的最小语法就是 `[title](src|mode=supertag)`。实际文档里，`src` 通常会换成目标文件路径。
</Note>

## 常见字段类型

| 类型 | 常见用途 |
| --- | --- |
| `text` | 标题、作者、备注、国家 |
| `number` | 年份、分数、排名 |
| `date` | 截止日期、观看日期、创建日期 |
| `checkbox` | 完成状态、暖色调、展示开关 |
| `multiselect` | 类型、标签、分类 |
| `array` | 主演列表、布局字段列表 |
| `progress` | 评分或进度类数值 |
| `asset` | 封面图和资源路径 |

## 推荐工作流

<Steps>
  <Step title="先复制一条已有记录">
    从字段结构最接近的文件开始，而不是每次从零手写。
  </Step>

  <Step title="保留字段结构和 class">
    维持相同的字段名、字段类型和 `class`，保证它还属于同一个集合。
  </Step>

  <Step title="替换具体值">
    再修改标题、日期、封面和正文内容。
  </Step>

  <Step title="统一补齐共享字段">
    如果整类记录都需要新增字段，后续再做一致性回填。
  </Step>
</Steps>

<Tip>
  SuperTag 最怕字段名频繁漂移。一个 class 一旦开始投入使用，就尽量保持字段稳定。
</Tip>
