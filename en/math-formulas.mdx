## Formula types

### Inline math

Inline math stays inside a text paragraph.

<AccordionGroup>
  <Accordion title="Markdown syntax" icon="code" defaultOpen>
    Wrap the expression with single dollar signs:

    ```markdown
    This is inline math $E = mc^2$.
    ```
  </Accordion>

  <Accordion title="Auto-conversion" icon="sparkles">
    Typing `$formula$` and pressing space or enter converts it into an inline math node.

    Rules include:
    - The delimiters must both be present
    - Empty `$$` does not convert into inline math
    - Dollar signs around plain currency should not be interpreted as formulas
  </Accordion>

  <Accordion title="Good use cases" icon="lightbulb">
    Use inline math for variables, short equations, limits, and compact notation inside normal prose.
  </Accordion>

  <Accordion title="Shortcut" icon="keyboard">
    - macOS: `Cmd + M`
    - Windows/Linux: `Ctrl + M`
  </Accordion>
</AccordionGroup>

<Tip>
  Keep inline formulas compact. If the formula includes large fractions, stacked scripts, or several lines, switch to display math.
</Tip>

### Display math

Display math uses a centered block that occupies its own line or lines.

<AccordionGroup>
  <Accordion title="Markdown syntax" icon="code" defaultOpen>
    Wrap the expression with double dollar signs:

    ```markdown
    $$
    \int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
    $$
    ```
  </Accordion>

  <Accordion title="Auto-conversion" icon="sparkles">
    Typing `$$` and pressing space creates a display-math editor block immediately.
  </Accordion>

  <Accordion title="Multi-line support" icon="lines-leaning">
    Display math supports aligned equations, matrices, piecewise functions, and other multi-line structures.
  </Accordion>

  <Accordion title="Shortcut" icon="keyboard">
    - macOS: `Cmd + Shift + M`
    - Windows/Linux: `Ctrl + Shift + M`
  </Accordion>
</AccordionGroup>

<Warning>
  Display math always occupies its own visual block. Use inline math when the expression should stay inside a sentence.
</Warning>

## Editing formulas

### Create a new formula

<Steps>
  <Step title="Use Markdown syntax">
    Type `$...$` for inline math or `$$...$$` for display math and let input rules convert it.
  </Step>

  <Step title="Use shortcuts">
    Insert an inline formula with `Cmd/Ctrl + M` or a display formula with `Cmd/Ctrl + Shift + M`.
  </Step>

  <Step title="Write LaTeX">
    Enter standard LaTeX math code inside the editor.
  </Step>

  <Step title="Finish editing">
    Press `Esc` or click outside the formula to render it.
  </Step>
</Steps>

### Edit an existing formula

<AccordionGroup>
  <Accordion title="Enter edit mode" icon="pen" defaultOpen>
    Click a rendered formula or move the cursor onto it and press `Enter`.
  </Accordion>

  <Accordion title="Update the source" icon="keyboard">
    Once inside edit mode, you can use normal text editing operations such as copy, paste, selection, and undo.
  </Accordion>

  <Accordion title="Exit edit mode" icon="door-open">
    Press `Esc` or click outside the formula to save and re-render it.
  </Accordion>

  <Accordion title="Navigation shortcuts" icon="arrows">
    Use arrow keys, `Home`, `End`, `Cmd/Ctrl + A`, `Backspace`, and `Delete` as expected while editing LaTeX source.
  </Accordion>
</AccordionGroup>

<Tip>
  If a formula fails to render, Zditor usually leaves the source visible or indicates an error. Check the LaTeX syntax first.
</Tip>

## LaTeX quick reference

### Basic operators

| Symbol | LaTeX | Meaning |
| --- | --- | --- |
| $+$ | `+` | Addition |
| $-$ | `-` | Subtraction |
| $\times$ | `\times` | Multiplication |
| $\div$ | `\div` | Division |
| $\pm$ | `\pm` | Plus or minus |
| $\cdot$ | `\cdot` | Dot product |
| $=$ | `=` | Equality |
| $\neq$ | `\neq` | Not equal |
| $\leq$ | `\leq` | Less than or equal |
| $\geq$ | `\geq` | Greater than or equal |

### Superscripts and subscripts

| Result | LaTeX | Meaning |
| --- | --- | --- |
| $x^2$ | `x^2` | Superscript |
| $x_i$ | `x_i` | Subscript |
| $x^{2y}$ | `x^{2y}` | Multi-character superscript |
| $x_{i,j}$ | `x_{i,j}` | Multi-character subscript |
| $x_1^2$ | `x_1^2` | Both at once |

<Tip>
  Use braces `{}` whenever a superscript or subscript contains more than one character.
</Tip>

| $\tfrac{x}{y}$ | `\tfrac{x}{y}` | Text-sized fraction |
| $\sqrt{x}$ | `\sqrt{x}` | Square root |
| $\sqrt[3]{x}$ | `\sqrt[3]{x}` | Nth root |

### Sums and integrals

| Result | LaTeX | Meaning |
| --- | --- | --- |
| $\sum_{i=1}^{n}$ | `\sum_{i=1}^{n}` | Summation |
| $\prod_{i=1}^{n}$ | `\prod_{i=1}^{n}` | Product |

### Matrices

```latex
$$
\begin{bmatrix}
1 & 2 \\
3 & 4
\end{bmatrix}
$$
```

Matrix rules:

- Use `&` to separate columns
- Use `\\` to separate rows
- Prefer display math for matrices

## Advanced techniques

<AccordionGroup>
  <Accordion title="Mix text inside formulas" icon="text">
    Use `\text{}` to insert normal text inside math mode.
  </Accordion>

  <Accordion title="Align multi-line equations" icon="align-justify">
    Use the `aligned` environment and place `&` at the alignment points.
  </Accordion>

  <Accordion title="Color formulas" icon="palette">
    Use `\color{}` for emphasis when color is supported.
  </Accordion>

  <Accordion title="Control spacing" icon="expand">
    Commands such as `\,`, `\:`, `\;`, `\quad`, and `\qquad` adjust spacing.
  </Accordion>

  <Accordion title="Tag equations" icon="list-ol">
    Use `\tag{}` to add manual numbering to a display equation.
  </Accordion>

  <Accordion title="Piecewise formulas" icon="bracket-curly">
    Use `cases` or matching `\left` and `\right.` constructs for piecewise expressions.
  </Accordion>
</AccordionGroup>

## FAQ

<AccordionGroup>
  <Accordion title="Why is my formula not rendering?" icon="question">
    Check for LaTeX syntax mistakes, unmatched delimiters, missing escapes, or a rendering error caused by an unsupported command.
  </Accordion>

  <Accordion title="How do I show a dollar sign?" icon="question">
    Escape it as `\$`, or place it inside `\text{}`.
  </Accordion>

  <Accordion title="Why is inline math stretching line height?" icon="question">
    Replace `\frac` with `\tfrac`, simplify the expression, or move it to display math.
  </Accordion>

  <Accordion title="How do I write aligned multi-line math?" icon="question">
    Use display math with an `aligned` environment and set `&` at alignment points.
  </Accordion>

  <Accordion title="Are all LaTeX commands supported?" icon="question">
    No. Zditor uses KaTeX, which covers most common math but not every package or feature such as TikZ.
  </Accordion>

  <Accordion title="How do I copy a formula?" icon="question">
    You can copy the LaTeX source from edit mode or copy the entire formula node and paste it elsewhere.
  </Accordion>
</AccordionGroup>

## Performance and best practices

<Steps>
  <Step title="Use formulas when they add value">
    Large numbers of formulas increase rendering cost, so keep them purposeful.
  </Step>

  <Step title="Keep inline math short">
    Use display math for anything complex.
  </Step>

  <Step title="Prefer semantic commands">
    Commands like `\sin` and `\log` render better than manually italicizing operator names.
  </Step>

  <Step title="Check rendering before saving">
    Catch syntax mistakes early by reviewing the rendered output.
  </Step>

  <Step title="Reuse common snippets">
    Save your most frequent formulas as templates or snippets.
  </Step>
</Steps>
