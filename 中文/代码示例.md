# 代码示例

这个文档用于展示在 Zditor 文档中编写代码示例的常见方式，包括行内代码、常见语言代码块、配置文件、接口请求和 diff 示例。

## 行内代码 Inline Code

可以在段落中直接插入行内代码，例如 `const app = "zditor"`、`npm run dev`、`docs.zditor.com`、`POST /api/documents`。

适合行内代码的内容包括：

- 命令：`git status`
- 变量名：`userId`
- 函数名：`renderMarkdown()`
- 配置项：`enableCache`
- 文件路径：`中文/代码示例.md`

## TSX / React

```tsx
import { useState } from "react";

type CounterProps = {
  initial?: number;
};

export function Counter({ initial = 0 }: CounterProps) {
  const [count, setCount] = useState(initial);

  return (
    <button onClick={() => setCount((value) => value + 1)}>
      Count: {count}
    </button>
  );
}
```

## Python

```py
def fibonacci(n: int) -> list[int]:
    values = [0, 1]
    while len(values) < n:
        values.append(values[-1] + values[-2])
    return values[:n]


print(fibonacci(8))
```

## SQL

```sql
SELECT
  author_id,
  COUNT(*) AS post_count,
  MAX(published_at) AS latest_post_at
FROM posts
WHERE status = 'published'
GROUP BY author_id
ORDER BY latest_post_at DESC;
```

## 代码块书写建议

- 为代码块加上明确的语言标记，例如 `js`、`ts`、`bash`、`json`
- 一段代码只表达一个重点，避免把太多上下文塞进同一个块
- 配置示例和执行命令建议分开写，阅读更清楚
- 较长代码块适合配一段简短说明，说明它的用途和关注点

!!! tip 推荐用法
    如果你在写教程，正文里用行内代码点名命令、变量和文件路径，再用代码块展示完整上下文，阅读体验会更稳定。
