## Overview

## Core Inline Marks

| Mark | Syntax | Typical use |
| --- | --- | --- |
| Bold | `**text**` | key conclusions and important terms |
| Italic | `*text*` | mild emphasis, foreign words, special terms |
| Highlight | `==text==` | important definitions and takeaways |
| Inline code | `` `code` `` | commands, variables, identifiers |
| Link | `[text](url)` | navigation and references |

<Tip>
  Keep formatting light. Text marks work best when they make structure clearer instead of competing for attention.
</Tip>

## Review Marks

<AccordionGroup>
  <Accordion title="Annotations with mode=tip" icon="lightbulb" defaultOpen>
    Use annotations when the original text is correct but needs more context.

    ```md
    [title](src|mode=tip)
    ```

    Expanded form:

    ```md
    [term](explanation%20goes%20here|mode=tip|style=teal)
    ```

    Typical use cases:

    - explain technical terms
    - add reading hints
    - attach short context without rewriting the sentence
  </Accordion>

  <Accordion title="Revisions with mode=revision" icon="pen-to-square">
    Use revisions when the current wording should be replaced.

    ```md
    [title](src|mode=revision)
    ```

    Expanded form:

    ```md
    [original text](why%20it%20should%20change|mode=revision|style=indigo|advice=replacement%20text)
    ```

    Typical use cases:

    - fix typos
    - tighten wording
    - replace vague dates with specific ones
  </Accordion>

  <Accordion title="When to encode spaces as %20" icon="code">
    If you write the raw Markdown source directly, spaces inside `tip` and `revision` parentheses should be encoded as `%20`.

    That applies to:

    - annotation text
    - file paths
    - URLs
    - `advice` values
  </Accordion>
</AccordionGroup>

## Suggested Color Meanings

| Color | Typical use |
| --- | --- |
| `red` | errors, removals, vague dates, spelling problems |
| `indigo` | wording improvements, tone cleanup, style alignment |
| `teal` | neutral explanations and general suggestions |
| `yellow` | warnings and extra attention |
| `green` | positive confirmation or keep-as-is guidance |

## Copy-Ready Templates

### Annotation template

```md
This sentence includes a [term](explain%20the%20meaning%20here|mode=tip|style=teal).
```

### Revision template

```md
Please change [the original phrase](explain%20why%20it%20should%20change|mode=revision|style=indigo|advice=the%20revised%20phrase).
```

<Warning>
  `revision` must include `advice`. Without it, the mark points out a problem but does not provide a replacement.
</Warning>
