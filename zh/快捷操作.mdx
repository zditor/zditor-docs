## 拖拽

### 支持的 5 类拖拽

Zditor 的拖拽不只是“文档内部重排”，而是覆盖了多种来源和目标。

#### 1. 文件树内部拖拽

左侧文件树中的**文件和文件夹**都支持在文件树内部直接拖拽，以调整它们在工作区中的位置。

这类拖拽适合：

- 调整文件夹层级
- 把文件移动到其他目录
- 整理工作区结构

#### 2. 从左侧文件树拖到 Markdown 文档

左侧文件树中的文件可以直接拖拽到 Markdown 文档中。

默认情况下，这些文件会先以**附件节点**的形式插入。插入后，点击该节点可以弹出工具栏，再把它转换成更合适的节点类型，例如：

- 图片节点
- 视频节点
- 其他更适合当前文件类型的展示形式

这个流程适合“先快速插入，再决定展示方式”。

#### 3. 从系统外部拖拽文件到 Markdown 文档

你也可以把系统文件管理器里的文件直接拖进 Markdown 文档。

系统会根据文件类型自动决定插入后的节点形式，常见情况包括：

- 图片文件 -> 图片节点
- 视频文件 -> 视频节点
- 其他文件 -> 附件节点

这种方式适合快速把素材从本地拖进笔记或文档。

#### 4. 在不同 Markdown 文档之间拖拽节点

不同 Markdown 文档之间也支持节点拖拽，例如左右分屏场景下，把一个文档里的节点拖到另一个文档中。

跨文档拖拽的默认行为是 **copy**，也就是复制一份到目标文档，而不是把原文档里的节点移走。

这类跨文档拖拽适合：

- 把草稿内容整理到正式文档
- 在左右分屏中搬运素材块
- 快速重组多篇笔记中的内容

#### 5. 文档内部位置拖拽

同一篇文档内部支持块级节点重排。你可以直接把一个节点拖到文档中的其他位置，而不需要先剪切再粘贴。

同一文档内部拖拽的默认行为是 **move**，也就是直接移动节点位置。

这也是最常见的一类拖拽，适合：

- 调整段落顺序
- 重新编排章节结构
- 把说明、图片、代码块移动到更合理的位置

### 编辑器中的拖拽重排

在编辑器中，把鼠标移动到块级内容附近时，会出现拖拽按钮。拖动这个按钮，可以直接重排内容块的位置，而不需要剪切、粘贴。

目前常见的节点类型基本都支持拖拽，例如：

- 段落
- 标题
- 列表
- 图片
- 视频
- 代码
- 附件
- Admonition
- 表格

<Warning>
  **拖拽异常反馈**

  如果你发现某一种节点拖拽时存在问题，例如：

  - 不能拖拽
  - 拖拽位置不对
  - 拖拽后内容异常

  可以直接提交 bug 反馈，我们会继续补齐和修复。
</Warning>

### 拖拽按钮通常出现在哪里

不同节点的拖拽按钮位置不完全一样，通常会出现在节点的一侧：

- **左侧**：例如段落这类普通块级内容
- **右侧**：例如列表、视频、图片等部分节点

这意味着你在使用时不一定总是在同一个位置找拖拽按钮，而是要根据节点类型观察它靠近哪一侧。

<Tip>
  拖拽按钮的核心用途是“重排内容结构”。如果你正在整理长文档、会议纪要或笔记，这个功能会比手动剪切粘贴快很多。
</Tip>

### 拖拽时会发生什么

拖拽时，系统会显示一个带图标的拖拽提示，帮助你确认当前移动的内容类型，例如段落、标题或代码块。

这类拖拽更适合：

- 调整章节顺序
- 重新组织列表结构
- 把一段说明移到更合适的位置
- 快速整理草稿

## 快捷键

快捷键是 Zditor 里最直接的提效方式之一。常见场景包括：

- 保存文档
- 搜索内容
- 设置标题级别
- 切换粗体、斜体、高亮、行内代码
- 创建列表、引用块、表格、代码块、数学公式

### 最常用的一批快捷键

| 快捷键 | 功能 |
| --- | --- |
| `Mod-K` | 打开搜索 |
| `Mod-S` | 保存当前文档 |
| `Mod-B` | 切换粗体 |
| `Mod-I` | 切换斜体 |
| `Mod-H` | 切换高亮 |
| `Mod-Shift-C` | 切换行内代码 |
| `Mod-1` ~ `Mod-6` | 设置标题级别 |
| `Mod-Shift-8` | 切换无序列表 |
| `Mod-Shift-7` | 切换有序列表 |
| `Mod-Shift-9` | 切换复选框列表 |
| `Mod-Shift-\`` | 切换代码块 |
| `Mod-T` | 创建表格 |

<Warning>
  `Mod` 在不同系统上含义不同：
  <ul>
    <li><strong>macOS</strong>：`Command`</li>
    <li><strong>Windows / Linux</strong>：`Ctrl`</li>
  </ul>
</Warning>

### 什么时候优先用快捷键

快捷键尤其适合下面这些场景：

- 你已经知道自己要插入什么格式
- 你在整理大量文本，不想频繁切换鼠标
- 你在做重复性格式化

如果你需要完整列表，请直接查看 [快捷键指南](/zh/快捷键指南)。

## 快捷命令

快捷命令主要用于输入框中的结构化操作。

### `/` 命令菜单

在支持命令输入的输入框中，输入 `/` 会弹出命令菜单。你可以继续输入关键词进行过滤，然后选择一条命令插入到输入框中。

根据不同模式，这个列表的来源会不同：

- **Chat 模式**：优先展示可复用的提示词
- **Agent 模式**：展示当前可用的 Agent 命令

命令菜单的作用是：

- 减少重复输入
- 避免记不住完整命令
- 通过列表快速发现当前可用能力

### 适合使用快捷命令的场景

- 想快速调用常用提示词
- 想查看 Agent 当前支持的命令
- 想用结构化方式发起任务，而不是每次都手写完整指令

<Tip>
  如果你经常使用同一类 AI 操作，优先用 `/` 命令会比自然语言重复输入更稳定。
</Tip>

### 补充：编辑器里的其他触发式菜单

除了 `/` 之外，编辑器中还有一些常见的触发式菜单：

- `@`：引用工作区中的 Markdown、PDF 等资源
- `#`：插入或创建 SuperTag

如果你正在做知识整理或文档互链，这两个入口会非常常用。

## 工具栏

ZDitor 的工具栏不是单一固定的一条，而是“按上下文变化”的。

### 选择工具栏

在编辑器里选中文字或节点后，会根据当前选中对象弹出对应工具栏。例如：

- 文本格式工具栏
- 链接工具栏
- 图片工具栏
- 脚注工具栏
- 表格工具栏
- SuperTag 工具栏
- PDF Card 工具栏
- 代码块工具栏
- 附件工具栏

这意味着你通常不需要先记住所有语法，而是可以：

1. 先选中内容
2. 再用工具栏完成调整

### 工具栏的特点

<AccordionGroup>
  <Accordion title="按对象变化" icon="arrows-rotate" defaultOpen>
    不同对象会出现不同工具栏。选中文本看到的是格式工具栏，选中图片看到的是图片工具栏，选中 PDF Card 则会弹出 PDF Card 工具栏。
  </Accordion>

  <Accordion title="支持二级浮层" icon="layer-group">
    某些工具栏按钮不仅能直接执行动作，还会打开一个二级浮层，用来选择更具体的设置或目标。
  </Accordion>

  <Accordion title="就地编辑" icon="pen-to-square">
    工具栏通常出现在选中内容附近，避免频繁把视线移到页面其他位置。
  </Accordion>
 </AccordionGroup>

### 什么时候优先用工具栏

工具栏更适合下面这些情况：

- 你知道要改哪块内容，但不想记语法
- 你想先看到可选动作，再决定怎么处理
- 你处理的是图片、附件、PDF Card、SuperTag 这类可视化节点

## 推荐的上手顺序

如果你刚开始使用 Zditor，推荐按下面的顺序熟悉：

1. 先学会拖拽重排内容结构
2. 再记住最常用的一批快捷键
3. 接着尝试 `/`、`@`、`#`、`:` 这类触发式菜单
4. 最后在实际写作中熟悉不同对象的工具栏
