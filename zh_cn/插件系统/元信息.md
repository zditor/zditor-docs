# 元信息（MetaData）

## 概述

元信息是 Zditor 中用于在文章顶部展示文章基本信息的组件，包括标题、副标题、作者、创建时间、标签等。元信息以 [Front Matter](https://jekyllrb.com/docs/front-matter/) 格式存储在文档开头。

## 支持的字段类型

Zditor 的元信息系统支持多种字段类型，可以满足不同场景的需求。

### 基础字段类型

|类型 |说明 |示例值 |
|---|---|---|
|`text` |单行文本 |`"文章标题"` |
|`asset` |文件资源路径 |`"./images/avatar.jpg"` |
|`date` |日期 |`"2024-01-15"` |
|`datetime` |日期时间 |`"2024-01-15T10:30:00"` |
|`time` |时间 |`"10:30:00"` |
|`array` |字符串数组 |`["标签1", "标签2"]` |
|`checkbox` |布尔值 |`true` |
|`select` |单选下拉框 |`"选项1"` |
|`multiselect` |多选下拉框 |`["选项1", "选项2"]` |
|`number` |数字 |`42` |
|`progress` |进度值（0-100） |`75` |

### 常用预定义字段

|字段名 |建议类型 |说明 |示例 |
|---|---|---|---|
|`title` |text |文章标题 |`title: 我的第一篇文章` |
|`subtitle` |text |文章副标题 |`subtitle: 一篇技术分享` |
|`authors` |array |作者列表 |`authors: ["张三", "李四"]` |
|`avatar` |asset |作者头像 |`avatar: ./images/avatar.jpg` |
|`created` |datetime |创建时间 |`created: 2024-01-15T10:30:00` |
|`tags` |array |标签列表 |`tags: ["技术", "分享"]` |
|`cover` |asset |封面图片 |`cover:\n  type: asset\n  value: ./images/bg.jpg` |
|`warm` |checkbox |推荐内容 |`warm: true` |
|`display` |checkbox |显示控制 |`display: true` |

## 基础用法

![元信息示意图](assets/截屏2025-11-02_19.54.18.png)

### 简单格式（类型推断）

Zditor 会自动推断字段类型，支持简化的 YAML 格式：

```yaml
---
title: 文章标题
subtitle: 文章副标题
authors: [张三, 李四]
created: 2024-01-15T10:30:00
tags: [技术, 分享, 前端]
cover:
  type: asset
  value: ./images/cover.jpg
avatar:
  type: asset
  value: ./images/avatar.jpg
warm: true
display: true
---
```

### 完整格式（显式类型定义）

对于复杂需求，可以显式定义字段类型和属性：

```yaml
---
title:
  type: text
  label: 文章标题
  value: 我的文章
  default: 未命名文章
  description: 请输入文章标题

authors:
  type: array
  label: 作者
  value:
    - 张三
    - 李四
  default: []
  description: 文章作者列表

created:
  type: datetime
  label: 创建时间
  value: 2024-01-15T10:30:00
  default: now
  description: 文章创建时间

priority:
  type: select
  label: 优先级
  value: high
  default: medium
  options: [low, medium, high, urgent]
  description: 文章优先级

progress:
  type: progress
  label: 完成进度
  value: 75
  default: 0
  description: 文章完成百分比
---
```

### 混合格式

可以在同一个文档中混合使用简单格式和完整格式：

```yaml
---
title: 混合格式示例  # 简单格式
authors: [张三]       # 简单格式

custom_field:        # 完整格式
  type: select
  label: 自定义字段
  value: option1
  default: option1
  options: [option1, option2, option3]
  description: 这是一个自定义字段
---
```

## 详细字段说明

### 基础字段类型详解

#### text（文本）

- **用途**: 单行文本输入
- **属性**:
  - `value`: 当前值（字符串）
  - `default`: 默认值
  - `label`: 显示标签
  - `description`: 字段描述
- **示例**:

  ```yaml
  title: 文章标题                    # 简单格式
  # 或完整格式
  title:
    type: text
    value: 文章标题
    default: 未命名文章
    label: 标题
    description: 请输入文章标题
  ```

#### asset（资源文件）

- **用途**: 文件路径，支持图片、文档等资源
- **属性**: 同 text 类型
- **支持格式**: jpg, png, gif, pdf 等多种文件类型
- **路径类型**: 相对路径或绝对路径
- **示例**:

  ```yaml
  avatar: ./images/avatar.jpg
  cover:
    type: asset
    value: ./assets/cover.jpg
  ```

#### date/datetime/time（日期时间）

- **date**: 仅日期（YYYY-MM-DD）
- **datetime**: 日期时间（ISO 8601 格式）
- **time**: 仅时间（HH:mm:ss）
- **特殊默认值**: `now` 表示当前时间
- **示例**:

  ```yaml
  created: 2024-01-15T10:30:00      # 自动推断为 datetime
  publish_date: 2024-01-15          # 自动推断为 date
  reminder_time: "14:30:00"         # 自动推断为 time
  ```

#### array（数组）

- **用途**: 字符串数组
- **支持格式**:
  - YAML 数组语法
  - JSON 数组字符串
  - 逗号分隔字符串
- **示例**:

  ```yaml
  tags: [技术, 分享, 前端]           # YAML 数组
  tags: "[\"技术\", \"分享\"]"        # JSON 字符串
  tags: "技术, 分享, 前端"           # 逗号分隔
  ```

#### checkbox（复选框）

- **用途**: 布尔值开关
- **值**: `true`/`false`
- **示例**:

  ```yaml
  published: true
  featured: false
  ```

#### select/multiselect（选择器）

- **select**: 单选
- **multiselect**: 多选
- **必需属性**: `options` 选项列表
- **示例**:

  ```yaml
  priority:
    type: select
    value: high
    options: [low, medium, high, urgent]
  
  categories:
    type: multiselect
    value: [技术, 教程]
    options: [技术, 教程, 分享, 新闻]
  ```

#### number（数字）

- **用途**: 数值输入
- **支持**: 整数、浮点数
- **示例**:

  ```yaml
  version: 1.2
  count: 42
  ```

#### progress（进度）

- **用途**: 进度百分比（0-100）
- **显示**: 进度条形式
- **示例**:

  ```yaml
  progress: 75
  completion_rate: 60
  ```

### 常用预定义字段

#### title（标题）

- **建议类型**: `text`
- **说明**: 文章的主要标题，显示在元信息区域的中上部
- **特点**: 支持中英文，超长时自动换行

#### subtitle（副标题）

- **建议类型**: `text`
- **说明**: 文章的副标题，标题下方显示
- **特点**: 可以为空，用于补充说明文章主题

#### authors（作者列表）

- **建议类型**: `array`
- **说明**: 文章作者列表，支持多个作者
- **显示规则**:
  - 第一个作者显示在用户信息区域
  - 超过 1 个作者时，最多显示到第 3 个作者，其余显示为 "等X人"

#### avatar（作者头像）

- **建议类型**: `asset`
- **说明**: 作者头像图片路径
- **特点**:
  - 当设置时，显示头像图片
  - 未设置但有作者信息时，显示作者姓名的首字母

#### created（创建时间）

- **建议类型**: `datetime`
- **格式**: ISO 8601 标准
- **特殊值**: `now` 表示当前时间
- **显示**: 格式化为本地化的日期格式

#### tags（标签）

- **建议类型**: `array`
- **说明**: 文章标签列表，用于分类和检索
- **显示**: 以 `#标签名` 的形式显示，胶囊样式

#### cover（封面图片）

- **建议类型**: `asset`
- **说明**: 封面图片路径或 URL
- **特点**: 图片自动适配，带渐变遮罩保证文字可读性

### 特殊字段

除了基本的元信息字段外，Zditor 还支持一些特殊的结构字段，用于控制元信息的展示和布局：

#### row（行布局）

- **类型**: `array`
- **说明**: 定义元信息展示的行结构，控制哪些字段显示在同一行
- **用途**: 自定义元信息的布局排列
- **示例**:

  ```yaml
  row: ["createdAt", "title", "subtitle"]  # 这些字段会显示在同一行
  ```

#### col（列布局）

- **类型**: `array`
- **说明**: 定义元信息展示的列结构，控制哪些字段显示在同一列
- **用途**: 自定义元信息的布局排列
- **示例**:

  ```yaml
  col: ["avatar", "tags"]  # 这些字段会显示在同一列
  ```

#### favorite（收藏）

- **类型**: `checkbox`
- **说明**: 标记文章为收藏状态
- **用途**: 快速识别重要的文章
- **示例**:

  ```yaml
  favorite: true
  ```

#### warm（推荐）

- **类型**: `checkbox`
- **说明**: 标记文章为推荐内容
- **用途**: 标识特别推荐的文章
- **示例**:

  ```yaml
  warm: true
  ```

#### display（显示控制）

- **类型**: `checkbox`
- **说明**: 控制是否在特定界面显示此元信息
- **用途**: 控制元信息的可见性
- **示例**:

  ```yaml
  display: true  # 显示在展示界面
  ```

#### template（模板）

- **类型**: `checkbox`
- **说明**: 标记当前文档为模板
- **用途**: 将文档作为模板使用
- **示例**:

  ```yaml
  template: true
  ```

## 高级用法

### 自定义字段

除了预定义字段外，还可以创建任意自定义字段：

```yaml
---
# 自定义字段示例
difficulty:
  type: select
  label: 难度级别
  value: intermediate
  default: beginner
  options: [beginner, intermediate, advanced, expert]
  description: 文章的难度级别

estimated_read_time:
  type: number
  label: 预计阅读时间（分钟）
  value: 15
  default: 5
  description: 读者完成阅读的大概时间

related_links:
  type: array
  label: 相关链接
  value:
    - "https://example.com/article1"
    - "https://example.com/article2"
  default: []
  description: 相关文章或资源的链接

review_status:
  type: multiselect
  label: 审核状态
  value: [技术审核]
  default: []
  options: [待审核, 技术审核, 内容审核, 已通过]
  description: 文章的审核流程状态
---
```

### 类型推断规则

Zditor 自动推断类型的规则：


1. **布尔值**: `"true"`/`"false"` → `checkbox`
2. **数字**: 可解析的数字字符串 → `number`
3. **数组**:
   - 以 `[` 开头的 JSON 数组 → `array`
   - 逗号分隔的字符串 → `array`
4. **日期时间**: ISO 8601 格式 → `datetime`
5. **日期**: YYYY-MM-DD 格式 → `date`
6. **时间**: HH:mm:ss 格式 → `time`
7. **其他**: 默认为 `text`

### 特殊值处理

- **空字符串**: 转换为对应类型的空值
- `null`: 转换为默认值
- `now`: 日期时间字段的特殊值，表示当前时间
- **数组中的混合类型**: 自动转换为字符串表示

## 编辑功能

### 悬停工具栏

当鼠标悬停在元信息区域时，会显示工具栏，包含以下功能：

- **编辑元数据**: 打开元数据编辑模态框
- **更换背景**: 打开图片选择器，更换背景图片
- **删除元信息**: 移除当前文章的元信息

### 编辑方式


1. **直接编辑 Front Matter**: 在文档开头修改 YAML 内容
2. **使用工具栏**: 悬停后点击编辑按钮，通过图形界面编辑

## 样式和主题

元信息组件采用现代化的设计风格：

- **背景**: 可选的背景图片，带有渐变遮罩
- **文字层次**:
  - 创建时间：次要信息，较小字号，浅色
  - 标题：主要标题，大字号，粗体
  - 副标题：辅助信息，常规字号，半透明
  - 作者信息：中等字号
  - 标签：胶囊样式，带有 `#` 符号
- **布局**: 分为上下两部分
  - 上半部分：创建时间、标题、副标题
  - 下半部分（底部模糊区域）：作者信息和标签
- **交互效果**:
  - 悬停时显示工具栏
  - 标签悬停时轻微放大
  - 图片懒加载

## 注意事项


1. **Front Matter 格式**: 元信息必须位于文档开头，使用 `---` 包围
2. **文件路径**: 使用相对路径时，路径相对于当前文档所在目录
3. **时间格式**: 建议使用 ISO 8601 格式，确保跨平台兼容性
4. **作者信息**: `authors` 和 `avatar` 可以配合使用，也可以单独使用
5. **性能**: 背景图片和头像采用懒加载方式，不影响页面初始加载速度
6. **响应式**: 自动适配不同屏幕尺寸，移动端友好
